●ブランチ
変更履歴を枝のように分岐して残していくための機能
あるブランチをもとに新しいブランチを作成することを ブランチを切る というので覚えておいてください
masterブランチとはリポジトリで最初のcommitを行なったときに作成されるブランチ

●別のブランチの作業内容(変更履歴)をブランチに取り込むことを merge (マージ) という

｜ーmasterブランチ
｜　このブランチをもとにタスクごとに作業用ブランチを切って、各ブ
｜　ランチの作業が完了したらその履歴を取り込むブランチ。
｜　masterブランチで作業を行なったり、機能を実装することはない。
｜ー各ブランチ
　　タスクごとに各作業を行うブランチ。
　　作業が完了したらその履歴をmasterに反映させる。

・ブランチを切りかた：作業をわけるやり方
git branch
* master  ←自分のブランチがmasterブランチという表示がでる。

「working」という名前の作業用ブランチを切ります
git branch [新しいブランチの名前]

・ブランチの切り替え方
　git checkout [切り替えたいブランチ名]
